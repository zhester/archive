CSS2 Rounded Box

For browsers that provide strong CSS2 support and PNG-24 rendering, here is a method to provide a somewhat stylish rounded box based on a very primitive document fragment.

Document Fragment

For this to work, we need to use two semantically-null elements. First, we need to know what parts of the document will be included in the box. This is the surrounding <div>. It's also convenient to use this <div> tag as the root for selectors. Second, we need to allow any kind and any number of interior block elements in the body of the box. Thus, we need another document delimiter (the interior <div> tag). Since these demarcations have no relevance to the information, it's okay to use semantically-null elements. The regions where the information needs to be marked, are the heading and any body elements (which remain unknown to this style sheet).

Future Plans

It will always be a dream of mine to allow this same document fragment to be styled with the maximum versatility:

  • The corner images must be placed as independent images.
  • The other images must tile on all four edges (removing the maximum width limitation).
  • The exterior areas of the box (where the rounding leaves space "behind" the box) must be transparent to allow the exterior background (or content) to remain visible. The current solutions rely on the graphics' backgrounds to cover up parts of other graphics.
  • After all that, it might be nice to use some actual background images to pretty-up the interior of the box.
